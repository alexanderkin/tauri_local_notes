# 第 14 步：给清空操作增加确认步骤

## 1. 这一步的目标

在执行“清空全部笔记”之前，先弹出确认提示，避免误操作。

## 2. 你刚刚做了什么

你修改了：

- `tauri-notes/src/main.js`

你完成的事情是：

- 点击“清空全部笔记”时先弹出确认框
- 点击“取消”时不执行清空
- 点击“确定”时才真正调用 Rust 清空文件

## 3. 这一步里最关键的代码

```javascript
const confirmed = window.confirm("确定要清空全部笔记吗？这个操作不能撤销。");

if (!confirmed) {
  message.textContent = "已取消清空操作。";
  return;
}
```

可以先这样理解：

- `window.confirm(...)`：弹出一个“确定 / 取消”的确认框
- 如果用户没有确认，就立刻结束这次操作

## 4. 为什么这一步重要

因为“清空全部笔记”属于危险操作。

这类操作如果没有保护，用户误点一次，就可能丢掉数据。

所以真实产品里，删除、清空、覆盖等操作通常都会增加确认步骤。

## 5. 这一步完成后你学到了什么

- 知道如何在前端做一个最简单的确认步骤
- 知道危险操作要优先考虑防误触
- 知道好的产品不仅要能做事，还要尽量避免用户出错

## 6. 这一步的小复盘

你刚刚做的事情是：
为清空操作增加了保护。

它在整个项目里的作用是：
提高应用的安全性和可靠性。

这一步最需要记住的是：

- 危险操作前先确认
- 取消时及时结束，不再继续执行
