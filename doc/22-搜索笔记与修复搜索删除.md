# 第 22 步：搜索笔记与修复搜索结果删除

## 1. 这一步的目标

给笔记列表增加搜索功能，并保证在搜索结果中删除笔记时，不会删错。

## 2. 你刚刚做了什么

你修改了：

- `tauri-notes/src/index.html`
- `tauri-notes/src/styles.css`
- `tauri-notes/src/main.js`

你完成的事情是：

- 页面新增了搜索输入框
- 输入关键词时，笔记列表会实时筛选
- 可以按标题或内容搜索
- 修复了搜索结果里删除可能删错的问题

## 3. 搜索功能是怎么实现的

前端先用：

```javascript
allNotes = JSON.parse(result);
```

保存全部笔记。

然后搜索时使用：

```javascript
filter(...)
```

从全部笔记里筛选标题或内容包含关键词的笔记。

可以先这样理解：

- `allNotes`：所有笔记
- `filteredNotes`：搜索后剩下的笔记

## 4. 为什么搜索后删除可能删错

原来删除按钮使用的是当前显示列表里的编号。

没有搜索时：

- 显示列表就是全部笔记
- 编号是正确的

搜索后：

- 显示列表只是部分笔记
- 编号会重新从 0 开始
- 这个编号不一定等于它在全部笔记里的真实位置

所以如果直接删除，可能删错。

## 5. 最终怎么修复

修复思路是：

不要使用搜索结果里的临时编号。

而是通过：

```javascript
allNotes.findIndex(...)
```

回到全部笔记里，找到这条笔记的真实编号。

这样前端传给 Rust 的 `index` 就是正确的。

## 6. 这一步完成后你学到了什么

- 前端可以先保存全部数据，再做筛选显示
- `filter` 可以从列表里筛选符合条件的数据
- 搜索结果列表不等于原始数据列表
- 删除这类操作必须使用原始数据里的真实编号

## 7. 这一步的小复盘

它在整个项目里的作用是：
让“搜索”和“删除”两个功能可以安全地组合使用。

这一步最需要记住的是：

- 显示列表和原始列表可能不是同一个列表
- 删除时要找真实编号
- 功能之间组合后，可能会出现新的边界问题
