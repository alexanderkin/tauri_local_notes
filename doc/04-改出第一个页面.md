# 第 4 步：改出第一个页面

## 1. 这一步的目标

把 Tauri 默认页面替换成我们自己的“本地笔记本首页”。

这一步先不做保存功能，只做页面结构。这样可以把学习难度拆开，不会一口气改太多内容。

## 2. 你刚刚做了什么

你修改了这个文件：

```text
tauri-notes/src/index.html
```

你把默认页面替换成了自己的笔记页面，加入了这些内容：

- 页面标题
- 标题输入框
- 内容输入框
- 保存按钮
- 提示信息区域

## 3. 这个文件的作用

`index.html` 是页面结构文件。

可以把它理解成：
页面里“有什么东西”，主要在这里定义。

例如：

- 标题文字
- 输入框
- 按钮
- 提示区域

这些都属于页面结构。

## 4. 这一步里出现的新标签

### `input`

表示单行输入框。

我们用它来输入笔记标题。

### `textarea`

表示多行输入框。

我们用它来输入笔记正文。

### `button`

表示按钮。

后面点击“保存笔记”时，就会触发相应功能。

### `p`

表示一段文字。

我们先用它显示提示信息，后面也可以拿它显示操作结果。

## 5. 为什么这一步只改页面，不加功能

因为当前阶段最重要的是：
先把“界面骨架”搭出来。

如果这一步同时加入：

- 页面结构
- 样式
- 交互
- Rust 调用
- 文件保存

那对于初学者来说会太乱，也很难定位问题。

所以我们故意分开：

1. 先让页面出现
2. 再让页面更好看
3. 再让按钮有反应
4. 再连接 Rust
5. 再做本地保存

## 6. 这一步完成后你学到了什么

- 知道 `index.html` 负责页面结构
- 知道 `input` 和 `textarea` 的基本作用
- 知道项目开发时要先拆小步骤，不要一口气把所有事情塞进同一步

## 7. 常见错误和排查方法

### 情况 1：页面空白

可能原因：

- HTML 标签没写完整
- 标签嵌套不正确
- 文件保存后格式被破坏

处理方法：
优先检查 `index.html` 是否有明显缺失的开始标签或结束标签。

### 情况 2：页面能显示，但布局很乱

这通常不是严重错误。

因为这一步我们主要是先改结构，样式会在下一步继续处理。

### 情况 3：修改后窗口没有变化

排查方向：

- 确认文件是否已经保存
- 看开发窗口是否自动刷新
- 必要时重新运行 `npm.cmd run tauri dev`

## 8. 这一步的小复盘

你刚刚做的事情是：
把默认页面换成了我们自己的应用页面。

它在整个项目里的作用是：
让这个项目第一次开始长成“你自己的产品”，而不是官方示例。

这一步最需要记住的是：

- `index.html` 决定页面里有什么
- 先搭结构，再加样式和功能
- 每次只改一小块，更容易理解，也更容易排错

## 9. 下一步要做什么

下一步我们会只做一件事：
修改样式文件，让这个页面更清楚、更像一个真正的小应用界面。
