# Tauri 项目学习路线总览

## 1. 我们要做什么项目

我们要做的是一个 **本地笔记本桌面 App**。

它的核心功能会很简单：

- 输入笔记标题
- 输入笔记内容
- 点击按钮保存
- 从本地文件里读取笔记
- 在页面上显示出来

这个项目适合新手，因为它不花哨，但很完整。做完以后，你会真正走完一遍桌面 App 的常见开发流程。

## 2. 为什么选这个项目

我们没有选太复杂的项目，也没有一上来用复杂前端框架。

这样设计的原因是：

- 你现在是零基础，先把最核心的流程跑通更重要
- Tauri 的重点不是炫技，而是先理解“页面”和“本地能力”怎么连接
- 这个项目刚好能覆盖 Tauri 最重要的入门内容

## 3. 整个学习路线

我们会按阶段推进，每次只做一小步。

### 阶段 1：环境准备

目标：
让电脑具备运行 Tauri 项目的基本条件。

这一阶段完成后，你会得到：

- 可用的 Node.js
- 可用的 npm
- 可用的 Rust
- Windows 编译工具

### 阶段 2：创建第一个 Tauri 项目

目标：
从零创建一个 Tauri 项目，并成功运行出桌面窗口。

这一阶段完成后，你会得到：

- 一个可以启动的 Tauri 项目
- 第一个桌面窗口
- 对项目目录结构的初步认识

### 阶段 3：做出第一个页面

目标：
把默认页面改成我们自己的“笔记本首页”。

这一阶段完成后，你会得到：

- 标题
- 输入框
- 文本区域
- 保存按钮

### 阶段 4：让前端调用 Rust

目标：
让页面按钮能调用 Rust 代码，并拿到返回结果。

这一阶段完成后，你会得到：

- 第一个 Tauri 命令
- 前后端通信
- 对 Tauri 工作方式的初步理解

### 阶段 5：把笔记保存到本地

目标：
用 Rust 把笔记写入本地文件，并支持读取。

这一阶段完成后，你会得到：

- 本地文件读写能力
- 一个真正可用的小工具

### 阶段 6：整理体验并打包

目标：
把应用收尾，并生成可以分发的桌面程序。

这一阶段完成后，你会得到：

- 可运行的安装包或可执行程序
- 对发布流程的基础理解

## 4. 我们会用哪些技术

### 前端：HTML + CSS + JavaScript

原因：

- 最容易上手
- 资料多
- 不需要额外先学前端框架
- 适合先理解最基础的页面结构和交互

### 桌面框架：Tauri

原因：

- 体积通常比较小
- 可以调用本地能力
- 很适合做跨平台桌面应用

### 后端语言：Rust

原因：

- Tauri 默认使用 Rust
- 我们只学项目中真正用到的部分
- 先会用，再逐步理解

## 5. 当前约定

- 每次只做一步
- 每一步都尽量可运行
- 每一步做完都要复盘
- 如果报错，先排错，再继续新内容
- 所有学习内容都整理到 `doc` 目录下
