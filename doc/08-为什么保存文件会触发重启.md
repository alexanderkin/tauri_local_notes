# 第 8 步：为什么保存文件会触发应用重启

## 1. 你遇到的现象

每次点击保存后，终端都会出现类似提示：

```text
Info File src-tauri\notes.txt changed. Rebuilding application...
```

然后应用会重新编译并重启。

## 2. 这不是代码写错

这属于开发模式下的正常现象，不是保存功能本身出错了。

原因是：

- 你现在用的是 `tauri dev`
- 开发模式会监视项目目录里的文件变化
- 你把笔记保存到了 `src-tauri/notes.txt`
- 开发工具看到 `src-tauri` 里的文件变化，就会以为你改了项目代码
- 所以它自动重新编译、重启应用

## 3. 问题的本质

这里混在一起了两类文件：

### 项目源码文件

这些文件是给开发工具看的，例如：

- `src-tauri/src/lib.rs`
- `src/index.html`
- `src/main.js`

### 应用运行数据

这些文件是程序运行时产生的，例如：

- 用户保存的笔记
- 配置
- 缓存

这两类文件最好不要放在同一个源码目录里。

## 4. 正确思路

更合理的做法是：

- 项目代码继续放在项目目录里
- 用户数据保存到“应用数据目录”

这样做的好处是：

- 保存笔记不会触发开发模式重启
- 更接近真实桌面应用的做法
- 将来打包后也更自然

## 5. 这一步的小复盘

你刚刚发现的不是小问题，而是一个很有价值的开发常识：

- 源码目录不适合长期存放运行时数据
- 开发模式会监视项目文件变化
- 用户数据应该放到应用自己的数据目录

## 6. 下一步要做什么

下一步我们会把 `notes.txt` 的保存位置，从 `src-tauri` 目录改到应用数据目录。
