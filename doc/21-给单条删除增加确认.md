# 第 21 步：给单条删除增加确认

## 1. 这一步的目标

在删除单条笔记之前，先弹出确认框，避免误删。

## 2. 你刚刚做了什么

你修改了：

- `tauri-notes/src/main.js`

你完成的事情是：

- 点击“删除这条”时，先弹出确认提示
- 点击“取消”时不删除
- 点击“确定”时才调用 Rust 删除对应笔记

## 3. 为什么这一步重要

删除属于危险操作。

真实产品里，删除、清空、覆盖这类操作，一般都需要额外保护。

否则用户误点一次，就可能造成数据损失。

## 4. 这一步里最关键的代码

```javascript
const confirmed = window.confirm("确定要删除这条笔记吗？这个操作不能撤销。");

if (!confirmed) {
  message.textContent = "已取消删除操作。";
  return;
}
```

可以先这样理解：

- `confirm`：先问用户确定吗
- 用户取消时，立刻 `return`
- 这样后面的删除代码就不会执行

## 5. 这一步完成后你学到了什么

- 单条删除也需要防误操作
- 危险操作前加确认，是很常见的产品设计
- `return` 可以提前结束当前操作

## 6. 这一步的小复盘

你刚刚做的事情是：
给单条删除加了一层安全保护。

它在整个项目里的作用是：
让应用在能管理数据的同时，也更不容易误删数据。

这一步最需要记住的是：

- 删除前确认
- 取消时 return
- 安全性也是产品体验的一部分
