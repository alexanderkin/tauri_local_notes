# 第 20 步：基于 JSON 删除单条笔记

## 1. 这一步的目标

给每条笔记增加“删除这条”按钮，让用户可以只删除某一条笔记。

## 2. 为什么 JSON 更适合做这个功能

以前用纯文本保存时，笔记之间靠分隔线区分。

如果想删除某一条，就要手动拆文本、找分隔线、再重新拼文本。

现在用 JSON 后，每条笔记都是列表中的一个对象。

所以删除某一条就变得更清楚：

```rust
notes.remove(index);
```

可以理解成：
从笔记列表里删除指定编号的那一项。

## 3. Rust 后端做了什么

你新增了：

```rust
delete_note
```

它接收一个 `index`，也就是要删除的笔记编号。

核心逻辑是：

1. 读取全部笔记
2. 检查编号是否存在
3. 删除对应笔记
4. 把新的列表写回 JSON 文件

## 4. 前端做了什么

前端在每张笔记卡片里新增了一个按钮：

```html
删除这条
```

并通过：

```javascript
data-index="${index}"
```

把当前笔记编号放到按钮上。

点击按钮时，再调用：

```javascript
invoke("delete_note", { index })
```

让 Rust 删除对应那一条。

## 5. 这一步完成后你学到了什么

- JSON 让“删除某一条数据”变得更简单
- 前端可以把编号藏在按钮属性里
- Rust 根据编号修改数据列表
- 删除后要重新刷新页面显示

## 6. 这一步的小复盘

你刚刚做的事情是：
让每条笔记都可以单独删除。

它在整个项目里的作用是：
让这个应用从“只能整体清空”升级成了“可以精细管理数据”。

这一步最需要记住的是：

- 列表里的第 1 条编号是 `0`
- `data-index` 可以保存当前按钮对应的编号
- `notes.remove(index)` 可以删除指定位置的数据
