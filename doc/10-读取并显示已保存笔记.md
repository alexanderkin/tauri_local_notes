# 第 10 步：读取并显示已保存笔记

## 1. 这一步的目标

让页面在打开时自动读取已经保存的笔记，并把内容显示出来。

这样项目就形成了一个更完整的闭环：

- 输入笔记
- 保存笔记
- 读取笔记
- 显示笔记

## 2. 你刚刚做了什么

你修改了这几个文件：

- `tauri-notes/src/index.html`
- `tauri-notes/src/styles.css`
- `tauri-notes/src/main.js`
- `tauri-notes/src-tauri/src/lib.rs`

你完成的事情是：

- 页面新增了“已保存的笔记”区域
- Rust 新增了读取文件的功能
- 前端在页面打开时会主动读取笔记
- 每次保存成功后，页面会重新刷新显示内容

## 3. 这一步里最关键的思路

### Rust 负责读文件

Rust 新增了一个命令：

```rust
load_notes
```

它的作用是：
找到笔记文件，并把文件内容读成一段文字返回给前端。

### 前端负责显示内容

前端新增了一个函数：

```javascript
refreshNotes()
```

它的作用是：
调用 Rust 的读取功能，然后把结果显示到页面上的 `notes-list` 区域。

## 4. 为什么用 `pre`

你在页面里新增了：

```html
<pre id="notes-list">这里以后会显示笔记内容。</pre>
```

`pre` 可以先理解成：
“按原样显示文字”的区域。

因为我们现在是直接把文本文件内容显示出来，所以它比普通段落更合适。

## 5. 这一步完成后你学到了什么

- 知道前端不仅能保存数据，也能主动读取数据
- 知道 Rust 可以把文件内容整段返回给前端
- 知道页面打开时可以自动执行一段初始化逻辑
- 知道保存后刷新显示，是很常见的页面交互方式

## 6. 常见错误和排查方法

### 情况 1：页面显示“读取笔记失败”

排查方向：

- 检查 `load_notes` 是否已经注册到 `invoke_handler`
- 检查前端调用名是否写成了 `"load_notes"`
- 检查 Rust 代码是否成功编译

### 情况 2：文件里有内容，但页面没更新

排查方向：

- 检查 `refreshNotes()` 是否真的被调用
- 检查 `notesList.textContent = result;` 是否存在
- 检查页面里是否有 `id="notes-list"`

### 情况 3：页面打开时没有任何显示

排查方向：

- 检查最下面是否调用了 `refreshNotes()`
- 检查 `pre` 区域是否正确加到了 `index.html`
