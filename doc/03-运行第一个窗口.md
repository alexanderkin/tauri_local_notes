# 第 3 步：运行第一个 Tauri 窗口

## 1. 这一步的目标

让项目真正运行起来，并看到第一个桌面窗口。

这一步很重要，因为它证明了一件事：
我们的项目不只是“文件已经创建好了”，而是真的可以启动。

## 2. 你刚刚做了什么

你在项目目录里执行了这些命令：

```powershell
cd c:\Users\Kin\Desktop\codex\tauri-notes
```

作用：
进入项目目录。

```powershell
npm.cmd install
```

作用：
安装项目运行所需要的依赖。

```powershell
npm.cmd run tauri dev
```

作用：
以开发模式启动 Tauri 桌面应用。

## 3. 什么叫“开发模式”

这里的“开发模式”可以先理解成：

- 适合边写边看效果
- 改完代码后，通常能更快看到变化
- 主要用于开发阶段，不是最终发布版本

所以现在我们运行的是“方便开发的版本”。

## 4. 看到窗口意味着什么

当桌面窗口成功弹出来时，说明下面几件事都已经基本正常：

- 前端页面可以运行
- Rust 部分可以编译
- Tauri 框架可以正常启动
- 电脑环境已经足以支撑后续开发

这一步是整个项目非常关键的里程碑。

## 5. 这一步为什么重要

因为从现在开始，我们不再只是搭环境、建项目。

我们已经进入真正的开发阶段了。

后面你做的每一次页面修改、每一次按钮点击、每一次调用 Rust，都会直接反映在这个窗口里。

## 6. 这一步完成后你学到了什么

- 知道如何安装项目依赖
- 知道如何启动 Tauri 项目
- 知道看到桌面窗口就说明项目主链路已经跑通

## 7. 常见错误和排查方法

### 情况 1：`npm.cmd install` 失败

可能原因：

- 网络问题
- 权限问题
- 依赖下载失败

处理方法：
把完整报错贴出来，优先排错。

### 情况 2：`npm.cmd run tauri dev` 很慢

第一次启动通常会比较慢，因为它可能需要编译 Rust 代码。

这是常见现象，不一定是错误。

### 情况 3：窗口没有弹出来

排查方向：

- 看终端是否有报错
- 看是否有 Rust 编译错误
- 看是否有系统工具缺失
