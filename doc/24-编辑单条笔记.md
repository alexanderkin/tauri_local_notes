# 第 24 步：编辑单条笔记

## 1. 这一步的目标

让用户可以编辑已经保存的某一条笔记。

编辑功能完成后，应用就不只是能新增、搜索、删除，还能修改已有内容。

## 2. 你刚刚做了什么

你完成了三部分：

- 点击“编辑”后，把笔记内容填回输入框
- Rust 增加 `update_note` 命令
- 保存按钮根据状态执行“新增”或“更新”

## 3. 前端编辑状态

前端新增了：

```javascript
let editingNoteId = null;
```

可以先理解成：

- `null`：当前不是编辑状态
- 有 ID：当前正在编辑某条笔记

点击“编辑”按钮后，前端会：

- 找到对应笔记
- 把标题填回标题输入框
- 把内容填回内容输入框
- 把保存按钮改成“更新笔记”
- 记录当前正在编辑的笔记 ID

## 4. Rust 更新逻辑

Rust 新增了：

```rust
update_note
```

它接收：

- `id`
- `title`
- `content`

然后根据 ID 找到对应笔记，更新标题和内容，并重新写回 JSON 文件。

## 5. 保存按钮为什么能同时处理新增和更新

保存按钮点击时，会先判断：

```javascript
if (editingNoteId) {
  // 更新
} else {
  // 新增
}
```

这就是一个按钮支持两种模式的关键。

更新成功后，要把状态重置：

```javascript
editingNoteId = null;
saveButton.textContent = "保存笔记";
```

这样应用就回到普通新增模式。

## 6. 这一步完成后你学到了什么

- 编辑功能需要先进入编辑状态
- 稳定 ID 是编辑功能的基础
- 同一个按钮可以根据状态执行不同逻辑
- 更新数据时，Rust 需要找到对应条目并写回 JSON

## 7. 这一步的小复盘

你刚刚完成的是：
让已有笔记可以被修改。

它在整个项目里的作用是：
让笔记应用从“只能新增和删除”升级成了“可以完整管理内容”。

这一步最需要记住的是：

- 编辑要记住当前正在编辑哪条数据
- 更新完成后要退出编辑状态
- 新增和更新虽然共用输入框，但背后的逻辑不同
