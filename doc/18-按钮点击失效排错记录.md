# 第 18 步：按钮点击失效排错记录

## 1. 现象

你在运行：

```powershell
npm run tauri dev
```

之后发现：

- 页面上的按钮都没有反应
- 点击“保存笔记”没有提示
- 点击“清空全部笔记”也没有弹窗

## 2. 一开始先排查了什么

我们先没有怀疑 Rust，而是优先检查了最直接相关的前端文件：

- `src/index.html`
- `src/main.js`
- `src/styles.css`

这样做的原因是：
按钮点击无效，最常见的原因通常在前端，而不是后端。

## 3. 排查过程中发现的问题

### 问题 1：文件里出现乱码

最开始发现：

- `index.html` 里中文乱码
- `main.js` 里中文乱码
- `lib.rs` 里字符串也乱码

这说明文件编码可能被改坏了。

但是后续继续排查后发现：
**按钮无效的直接原因，不只是乱码。**

### 问题 2：先做了最小测试

我们把 `main.js` 临时改成最简单的测试代码，只保留：

- 页面加载后改一行提示文字
- 点击按钮后改提示文字

结果测试有效。

这说明：

- `main.js` 能被加载
- 按钮元素能找到
- `addEventListener` 本身没问题

也就是说：
**按钮无效不是因为 HTML 结构坏了，也不是按钮本身坏了。**

## 4. 继续定位到更深一层

后面我们继续测试了 Tauri 初始化部分，发现页面提示：

```text
Tauri 初始化失败：TypeError: Cannot read properties of undefined (reading 'core')
```

这句错误非常关键。

它说明：

```javascript
window.__TAURI__.core
```

这里拿到的是 `undefined`。

也就是说，代码一执行到这里就中断了，后面的按钮事件自然都没有绑定上。

## 5. 根因是什么

根因是：

**当前项目的前端接入方式，和 `main.js` 里的 Tauri API 用法不匹配。**

你的项目当前是：

- 简单静态前端模式
- 没有使用 `@tauri-apps/api` 那套前端包导入方式

而 Tauri 2 默认配置里：

```json
"withGlobalTauri": false
```

这意味着：

- 默认不会把 API 挂到 `window.__TAURI__`

所以你原来的写法：

```javascript
const { invoke } = window.__TAURI__.core;
```

就会直接报错。

## 6. 最终怎么修好的

我们做了两件事。

### 第 1 件事：在 `tauri.conf.json` 里打开全局 Tauri API

在：

```text
src-tauri/tauri.conf.json
```

的 `app` 配置里加入：

```json
"withGlobalTauri": true
```

### 第 2 件事：`main.js` 使用全局写法

继续使用：

```javascript
const { invoke } = window.__TAURI__.core;
```

这次之所以能成功，是因为前一步已经把全局 API 打开了。

### 第 3 件事：彻底重启开发环境

这一步也很重要。

因为：

- `tauri.conf.json` 是配置文件
- 改完它之后，只刷新前端通常不够
- 需要彻底关闭当前 `tauri dev`，再重新启动

## 7. 这次排错说明了什么

这次问题表面上看像是：

- 按钮没反应

但真正原因并不是：

- 按钮标签坏了
- `addEventListener` 不工作

而是：

- Tauri API 初始化失败
- 脚本在更前面就中断了

这说明排错时不能只盯着“表面现象”，要一步步缩小范围。

## 8. 这次排错的正确思路

这次我们实际走的是这样一条路线：

1. 先看页面结构有没有明显损坏
2. 再做最小按钮测试
3. 确认脚本本身能不能执行
4. 再检查 Tauri API 初始化
5. 最后修正项目接入方式和配置

这个顺序是对的，因为它能把问题一步步缩小。

## 9. 你现在最需要记住的

- 按钮没反应，不一定是按钮本身的问题
- 如果脚本更前面报错了，后面的事件绑定就不会执行
- Tauri 前端 API 的使用方式，要和项目结构匹配
- 配置改动后，记得彻底重启开发环境

## 10. 一句话复盘

这次你修好的不是一个“小按钮问题”，而是一次完整的“前端脚本初始化失败”排错案例。
