# 第 16 步：第一次成功打包桌面应用

## 1. 这一步的目标

把项目从开发状态打包成真正可以分发的桌面应用。

## 2. 你刚刚做了什么

你执行了：

```powershell
npm run tauri build
```

这条命令会：

- 编译 Rust
- 准备前端资源
- 调用 Tauri 的打包流程
- 生成桌面应用安装包

## 3. 你遇到过什么问题

一开始打包失败了。

原因不是业务代码错，而是 Windows 的打包工具在处理中文 `productName` 时不稳定。

后来你把：

```json
"productName": "本地笔记本"
```

改成：

```json
"productName": "Local Notes"
```

窗口标题仍然保留中文，这样就顺利打包成功了。

## 4. 最终打包成功的结果

成功生成了两个安装包：

- `src-tauri\target\release\bundle\msi\Local Notes_0.1.0_x64_en-US.msi`
- `src-tauri\target\release\bundle\nsis\Local Notes_0.1.0_x64-setup.exe`

## 5. 这一步完成后你学到了什么

- `npm run tauri build` 是正式打包命令
- 开发能运行，不代表发布一定能直接成功
- 打包阶段常见问题可能来自工具链兼容，而不是业务代码
- 排错时先选最稳、最省事的方案很重要

## 6. 这一步的小复盘

你刚刚完成的，不只是“运行项目”，而是“产出可发布版本”。

它在整个项目里的作用是：
证明你已经走完了一条完整的桌面应用开发闭环。
