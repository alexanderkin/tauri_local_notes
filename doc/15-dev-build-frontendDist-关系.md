# 第 15 步：理解 `npm run dev`、`npm run build` 和 `frontendDist`

## 1. 这一步的目标

弄清楚开发阶段和发布阶段分别在使用什么，以及它们之间是什么关系。

## 2. 三个关键点

### `npm run dev`

作用：
启动前端开发服务。

可以先理解成：
为了开发方便，临时开一个本地页面服务，方便边改边看效果。

### `npm run build`

作用：
把前端源码整理成可发布的前端成品。

可以先理解成：
把你写在 `src` 里的代码，打包成适合正式发布的一份结果。

### `frontendDist`

作用：
告诉 Tauri 在打包桌面应用时，去哪里拿前端成品。

## 3. 它们之间的关系

开发时：

- 运行 `npm run dev`
- Tauri 通过 `devUrl` 连接开发服务

打包时：

- 运行 `npm run build`
- 前端成品通常输出到 `dist`
- Tauri 通过 `frontendDist` 去拿 `dist` 里的内容

## 4. 为什么 `src` 和 `dist` 不一样

`src`：
你平时写代码的地方，属于源码目录。

`dist`：
构建后的成品目录，更适合正式打包和发布。

## 5. 一句最重要的总结

开发时看的是“临时服务”，发布时看的是“前端成品”。

## 6. 这一步的小复盘

你刚刚理解的是：

- 开发流程
- 构建流程
- Tauri 打包时前端资源从哪里来

这会直接帮助你后面理解为什么要先 `build`，再 `bundle`。
