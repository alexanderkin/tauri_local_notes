# 第 27 步：自定义应用图标

## 1. 这一步的目标

给 Tauri 应用换成自己的图标。

这样打包后的应用不再使用默认图标，而是有自己的识别标志。

## 2. 你刚刚做了什么

你完成了这些事情：

- 创建 SVG 源图标
- 把 SVG 转成 PNG
- 用 Tauri 命令生成多平台图标文件
- 重新打包
- 处理 Windows 图标缓存问题

## 3. 图标文件放在哪里

源文件放在：

```text
tauri-notes/assets/app-icon.svg
tauri-notes/assets/app-icon.png
```

Tauri 实际打包使用的图标放在：

```text
tauri-notes/src-tauri/icons
```

其中包括：

- `32x32.png`
- `128x128.png`
- `128x128@2x.png`
- `icon.ico`
- `icon.icns`

## 4. 生成图标的命令

在项目目录执行：

```powershell
npm.cmd run tauri icon assets/app-icon.png
```

这条命令会根据 PNG 源图，生成 Tauri 需要的各种图标文件。

## 5. 配置文件里的图标配置

`tauri.conf.json` 里需要有类似配置：

```json
"icon": [
  "icons/32x32.png",
  "icons/128x128.png",
  "icons/128x128@2x.png",
  "icons/icon.icns",
  "icons/icon.ico"
]
```

这些路径是相对于：

```text
src-tauri
```

来说的。

## 6. 重新打包

图标生成后，需要重新打包：

```powershell
npm.cmd run tauri build
```

## 7. 为什么图标一开始没立刻变

最后确认原因是：

**Windows 图标缓存。**

也就是说：

- 项目图标已经生成
- 配置也存在
- 打包也成功
- 但 Windows 仍然暂时显示旧图标

重启电脑后，新图标就正常显示了。

## 8. 遇到图标不更新时怎么排查

可以按这个顺序检查：

1. `assets/app-icon.png` 是否正确
2. `src-tauri/icons/icon.ico` 是否是新图标
3. `tauri.conf.json` 是否配置了 `icon`
4. 是否重新执行了 `npm.cmd run tauri build`
5. 是否卸载旧版本后再安装新版本
6. 是否需要重启电脑刷新 Windows 图标缓存

## 9. 这一步完成后你学到了什么

- Tauri 图标不是只放一张图片就结束
- 源图标和打包图标是两类文件
- 图标文件生成后还要重新打包
- Windows 可能缓存旧图标，导致看起来没有变化
