# 第 25 步：增加取消编辑功能

## 1. 这一步的目标

给编辑流程增加“取消编辑”按钮，让用户进入编辑状态后可以自然退出。

## 2. 你刚刚做了什么

你修改了：

- `tauri-notes/src/index.html`
- `tauri-notes/src/main.js`

你完成的事情是：

- 页面新增了“取消编辑”按钮
- 普通新增状态下，这个按钮隐藏
- 点击“编辑”后，这个按钮显示
- 点击“取消编辑”后，恢复普通新增状态

## 3. 为什么需要取消编辑

如果用户点了编辑，但后来不想改了，就需要一个明确的退出方式。

否则用户只能：

- 手动清空输入框
- 或者刷新页面

这都不够自然。

## 4. 这一步里最关键的思路

你新增了：

```javascript
function exitEditMode() {
  editingNoteId = null;
  titleInput.value = "";
  contentInput.value = "";
  saveButton.textContent = "保存笔记";
  cancelEditButton.hidden = true;
  titleInput.focus();
}
```

这个函数专门负责退出编辑状态。

这样不管是：

- 更新成功后退出
- 用户主动取消编辑

都可以调用同一个函数。

## 5. `hidden` 是什么

`hidden` 可以先理解成：
让页面元素暂时隐藏。

例如：

```html
<button hidden>取消编辑</button>
```

按钮还在页面里，只是暂时不显示。

## 6. 这一步完成后你学到了什么

- 编辑状态需要有明确退出方式
- 多处重复逻辑可以放进一个函数
- `hidden` 可以控制元素显示和隐藏
- 好的交互流程要考虑用户反悔的情况

## 7. 这一步的小复盘

你刚刚完成的是：
让编辑流程更加完整。

它在整个项目里的作用是：
让用户可以进入编辑、提交更新，也可以取消编辑。

这一步最需要记住的是：

- 进入编辑状态要记录 ID
- 退出编辑状态要清空 ID
- 状态变化时，按钮文字和显示状态也要同步变化
