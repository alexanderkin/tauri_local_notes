# 第 1 步：环境准备

## 1. 这一步的目标

让电脑具备运行 Tauri 项目的基础环境。

如果这一步没做好，后面创建项目时就容易报错。所以这一步不是“杂事”，而是项目的地基。

## 2. 这一阶段需要的工具

### Node.js

可以先把它理解成：
前端开发工具运行时所需要的环境。

### npm

可以先把它理解成：
前端项目的工具管理员。后面安装依赖、启动前端项目时会用到它。

### Rust

可以先把它理解成：
Tauri 后端使用的语言环境。

### Windows C++ 编译工具

可以先把它理解成：
Windows 上构建桌面程序需要的一套底层工具。

## 3. 当前检查结果

在当前电脑上，之前检查到的情况是：

- `node` 已安装
- `npm` 可用，但在 PowerShell 中直接输入 `npm` 可能被脚本策略拦住
- `Rust` 当时还未安装

后续你已经补装了相关环境。

## 4. 关于 npm 的一个注意点

在你的 PowerShell 环境里，直接运行 `npm` 可能会遇到脚本执行策略错误。

所以当前阶段更稳妥的做法是：

```powershell
npm.cmd -v
```

也就是说，后面如果我们写：

```powershell
npm install
```

你可以优先写成：

```powershell
npm.cmd install
```

这样更不容易被 PowerShell 的脚本策略拦住。

## 5. 你应该完成的检查命令

### 检查 npm

```powershell
npm.cmd -v
```

作用：
确认 npm 可以正常使用。

### 检查 Rust 编译器

```powershell
rustc -V
```

作用：
确认 Rust 编译器已经安装成功。

### 检查 Cargo

```powershell
cargo -V
```

作用：
确认 Rust 的项目工具已经安装成功。

## 6. 这一步为什么要这样做

因为 Tauri 项目不是只靠页面代码就能跑起来。

它至少涉及两部分：

- 前端部分
- Rust 后端部分

而 Windows 上构建桌面应用时，还需要额外的系统编译工具。

所以，环境准备是桌面开发中非常重要的一步。

## 7. 这一步完成后你会学到什么

- 知道 Tauri 项目依赖哪些基础工具
- 知道为什么桌面项目的环境准备比普通网页项目多一些
- 知道在当前 PowerShell 环境下，优先使用 `npm.cmd`

## 8. 常见错误和排查方法

### 情况 1：`rustc -V` 提示找不到命令

排查方法：

1. 关闭当前 PowerShell 窗口
2. 重新打开一个新的 PowerShell 窗口
3. 再执行一次 `rustc -V`

如果还是不行，通常是安装路径没有生效。

### 情况 2：`npm` 提示脚本执行被禁止

排查方法：

优先改用：

```powershell
npm.cmd -v
```

或其他 `npm.cmd` 命令。

### 情况 3：后面创建 Tauri 项目时报 Windows 编译相关错误

排查方向：

- 检查 Visual Studio C++ Build Tools 是否安装
- 检查是否勾选了 `Desktop development with C++`

## 9. 这一步的小复盘

你刚刚完成的不是业务代码，而是开发环境搭建。

它在整个项目里的作用是：
让我们后面创建、运行、编译 Tauri 项目时，不会被缺工具的问题拦住。

这一步最需要记住的内容是：

- Tauri 需要前端工具和 Rust 工具一起配合
- 你的 PowerShell 环境里，优先使用 `npm.cmd`
- 报错先排环境，再继续写代码
