# 第 2 步：创建 Tauri 项目

## 1. 这一步的目标

用 Tauri 官方脚手架创建一个新的桌面应用项目。

这一步完成后，我们就不再是“准备环境”，而是正式拥有了一个可以继续开发的项目。

## 2. 你刚刚做了什么

你执行了下面这条命令：

```powershell
npm.cmd create tauri-app@latest
```

这条命令的作用是：
使用 Tauri 官方提供的工具，生成一个新的项目骨架。

你在创建过程中选择了这些选项：

- 项目名：`tauri-notes`
- 标识符：`com.kin.taurinotes`
- 包管理工具：`npm`
- UI 模板：`Vanilla`
- UI 风格：`JavaScript`

## 3. 这些选项分别是什么意思

### 项目名：`tauri-notes`

这是项目文件夹的名字。

后面你会在这个目录里继续开发：

```text
tauri-notes
```

### 标识符：`com.kin.taurinotes`

可以先把它理解成：
这个桌面应用的“内部身份证”。

以后打包时，它会出现在应用配置里。

### 包管理工具：`npm`

意思是：
后面安装前端依赖、运行前端相关命令时，我们用 npm。

### UI 模板：`Vanilla`

这里的 `Vanilla` 表示：
不用 Vue、React 这类前端框架，而是直接用最基础的 HTML、CSS、JavaScript。

这很适合当前阶段，因为可以先把页面基础和 Tauri 连接方式学明白。

### UI 风格：`JavaScript`

意思是：
我们先用 JavaScript，不额外引入 TypeScript。

这样规则更少，入门更轻。

## 4. 创建成功的标志

你已经看到当前目录下出现了：

```text
tauri-notes
```

这说明项目骨架已经创建成功。

## 5. 这一步为什么重要

因为从这一步开始，我们终于有了一个真实的 Tauri 工程。

后面的所有学习都会围绕这个项目展开，包括：

- 运行窗口
- 修改页面
- 调用 Rust
- 保存本地文件
- 打包发布

## 6. 这一步完成后你学到了什么

- 知道如何用官方脚手架创建 Tauri 项目
- 知道为什么我们先选 `Vanilla + JavaScript`
- 知道项目文件夹已经成功生成

## 7. 常见错误和排查方法

### 情况 1：创建命令联网失败

原因：
脚手架需要从网上下载模板。

排查方向：

- 检查网络
- 检查 npm 是否可用
- 检查是否被系统策略或安全软件拦住

### 情况 2：不知道选哪个模板

当前学习路线中，优先选择：

- `Vanilla`
- `JavaScript`
- `npm`

这样最简单，最适合先学基础。

## 8. 这一步的小复盘

你刚刚做的事情是：
亲手创建了一个 Tauri 项目。

它在整个项目里的作用是：
给我们提供一个正式的开发起点。

这一步最需要记住的是：

- `create-tauri-app` 是 Tauri 官方建项目工具
- 我们故意选择了最简单的前端方案
- 先跑通项目，再逐步理解每个目录和每段代码

等这个窗口跑起来以后，我们再开始认识项目结构。
