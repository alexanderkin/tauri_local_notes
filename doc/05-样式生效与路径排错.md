# 第 5 步：让样式真正生效，并学会排查路径问题

## 1. 这一步的目标

让我们写的 `styles.css` 真正作用到页面上，并顺手学会一次非常常见的前端排错方法。

## 2. 你遇到的问题是什么

你已经修改了：

```text
tauri-notes/src/styles.css
```

但是页面没有任何变化，终端也没有报错。

这说明问题不在“样式内容”，而在“样式文件有没有被正确加载”。

## 3. 最后确认的原因

你把 `index.html` 里的这一行改成：

```html
<link rel="stylesheet" href="styles.css" />
```

样式就生效了。

这说明之前的问题是：
样式文件路径写法不对。

## 4. 为什么这样改就生效

当前文件位置是：

```text
tauri-notes/src/index.html
```

样式文件位置是：

```text
tauri-notes/src/styles.css
```

也就是说，`index.html` 和 `styles.css` 在同一个文件夹里。

所以最直接、最容易理解的写法就是：

```html
<link rel="stylesheet" href="styles.css" />
```

可以把它理解成：
“去当前文件所在的文件夹里，找一个叫 `styles.css` 的文件”。

## 5. 这一步学到的排错思路

当页面没有变化，但终端也没有报错时，不要立刻怀疑所有代码都有问题。

可以按这个顺序排查：

1. 文件有没有保存
2. 改的是不是正确的文件
3. 这个文件有没有被项目真正加载
4. 路径写法对不对

这次你遇到的，就是第 3 和第 4 类问题。

## 6. 为什么这次不用着急改成别的加载方式

虽然有些项目会在 `main.js` 里导入 CSS，但当前阶段我们先用最直观的方式：

```html
<link rel="stylesheet" href="styles.css" />
```

原因是：

- 更适合新手理解
- 你一眼就能看出页面加载了哪份样式
- 当前项目完全够用

后面等你更熟了，再去了解不同加载方式的区别。

## 7. 这一步完成后你学到了什么

- 知道 HTML 可以通过 `<link>` 加载样式文件
- 知道文件路径写错时，页面可能不会报错，但样式不会生效
- 知道排错时先确认“有没有被正确加载”，比盲改样式更重要

## 8. 这一步的小复盘

你刚刚做的事情是：
排查并修正了样式文件路径。

它在整个项目里的作用是：
确保我们后面写的样式真的能显示出来。

这一步最需要记住的是：

- `index.html` 和 `styles.css` 在同一个目录时，可以直接写 `href="styles.css"`
- 页面没变化，不一定是代码内容错了，也可能是文件根本没加载到
- 报错排查要先缩小范围
