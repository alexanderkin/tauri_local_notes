# 第 19 步：把笔记保存格式升级为 JSON

## 1. 这一步的目标

把笔记保存方式从纯文本升级成 JSON。

这一步是后续功能的基础，比如：

- 删除单条笔记
- 搜索笔记
- 编辑笔记
- 按时间排序

## 2. 为什么要从纯文本改成 JSON

之前的纯文本格式类似：

```text
时间：2026-04-19 10:00:00
标题：今天学习 Tauri
内容：学会了保存文件
--------------------
```

这种格式适合入门，但不适合继续做复杂功能。

JSON 格式更清楚，类似：

```json
[
  {
    "time": "2026-04-19 10:00:00",
    "title": "今天学习 Tauri",
    "content": "学会了保存文件"
  }
]
```

程序可以很容易地知道：

- 哪个是时间
- 哪个是标题
- 哪个是内容

## 3. Rust 后端做了什么

你在 `lib.rs` 里定义了：

```rust
struct Note {
    time: String,
    title: String,
    content: String,
}
```

可以先理解成：
规定一条笔记应该有哪些字段。

然后你增加了两个辅助函数：

- `read_notes`：读取全部笔记
- `write_notes`：写入全部笔记

JSON 版本的保存流程是：

1. 读出全部笔记
2. 把新笔记加入列表
3. 把整个列表重新写回文件

## 4. 前端做了什么

Rust 返回的是 JSON 字符串。

前端通过：

```javascript
JSON.parse(result)
```

把 JSON 字符串变成 JavaScript 能操作的数据列表。

然后用：

```javascript
note.time
note.title
note.content
```

直接显示每条笔记的数据。

## 5. 这一步完成后你学到了什么

- JSON 比纯文本更适合保存结构化数据
- 前后端可以围绕同一种数据结构协作
- Rust 负责读写 JSON 文件
- 前端负责解析 JSON 并显示成卡片

## 6. 这一步的小复盘

你刚刚做的事情是：
把笔记数据升级成结构化保存。

它在整个项目里的作用是：
为后续更复杂、更真实的笔记功能打基础。

这一步最需要记住的是：

- 纯文本适合入门
- JSON 更适合继续扩展
- 程序处理结构化数据会更轻松
